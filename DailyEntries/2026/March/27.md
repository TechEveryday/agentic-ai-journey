## Context and Planning

**Reasoning:** The repository had grown to need better organization. Daily entries capture daily reflections and progress, but lessons capture non-obvious insights that shouldn't be forgotten. By separating these concerns, future agents can quickly reference both daily context AND hard-won discoveries without duplicating information.

### Technology Choices for Calculator

- **Implementation:** Vanilla JavaScript with no frameworks for simplicity and educational clarity

### 2. Calculator Implementation

**Location:** `/projects/calculator/`

#### Files Created

**`index.html`** - Calculator UI
- Standard calculator button layout (0-9, decimal, operators, clear, delete, equals)
- Display area for showing current input and results
- HTML5 semantic markup with data attributes for easy JavaScript targeting
- Responsive viewport meta tag for mobile compatibility

**`styles.css`** - Visual Design
- Modern gradient background (purple theme)
- Clean, intuitive button layout with visual feedback
- Color coding:
  - Blue operators (+, −, ×, ÷)
  - Green equals button
  - Red clear button
  - Yellow delete/backspace button
  - Light gray number buttons
- CSS Grid for responsive button layout
- Hover and active states for user feedback
- Media queries for mobile devices

**`calculator.js`** - Core Logic
- **Calculator Class** - Encapsulates all calculator state and behavior
- **State Management:**
  - `currentValue` - Currently displayed number
  - `previousValue` - Number stored from previous operation
  - `operation` - Current pending operation (+, −, ×, ÷)
  - `shouldResetDisplay` - Flag to determine if next number input starts fresh
- **Key Methods:**
  - `handleNumber()` - Process number/decimal input with validation
  - `handleOperator()` - Handle operation selection and chaining
  - `calculate()` - Execute pending operation
  - `clear()` - Reset calculator to initial state
  - `delete()` - Remove last digit (backspace)
  - `updateDisplay()` - Sync display with internal state

#### Implementation Highlights

**1. Floating Point Precision**
```javascript
formatResult(num) {
    return Math.round(num * 100000000) / 100000000;
}
```
Addresses JavaScript's floating point arithmetic quirks (e.g., 0.1 + 0.2 = 0.30000000000000004).

**2. Decimal Point Validation**
```javascript
if (num === '.' && this.currentValue.includes('.')) {
    return;  // Prevent multiple decimal points
}
```
Ensures valid number format.

**3. Operation Chaining**
```javascript
if (this.operation !== null && !this.shouldResetDisplay) {
    this.calculate();  // Auto-calculate before new operation
}
```
Allows consecutive operations (5 + 3 + 2 evaluates as (5+3)+2 = 10) rather than requiring equals between operations.

**4. Division by Zero**
```javascript
result = current !== 0 ? prev / current : 0;
```
Gracefully handles division by zero.
