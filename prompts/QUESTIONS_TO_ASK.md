# Questions I Should Ask Before Starting a Project

Use this as a guide for clarifying requirements before we start coding.

## Requirements Clarity

- [ ] What is the primary goal of this project?
- [ ] Who are the users/stakeholders?
- [ ] What are the success metrics?
- [ ] Are there any non-negotiable constraints? (timeline, budget, etc.)

## Technical Decisions

- [ ] Which language/framework would you prefer?
  - Backend: C#/.NET, Node.js, Python, Go?
  - Frontend: React, Vue, Angular, or none?
  - Database: SQL (Postgres), NoSQL, other?
- [ ] Is there a specific UI library preference? (Material-UI, Tailwind, etc.)
- [ ] Authentication/Authorization requirements?
- [ ] Any integrations with external services?

## Architecture & Design

- [ ] How should we structure this following Clean Architecture?
- [ ] Are there specific design patterns you want applied?
- [ ] Monolith or microservices?
- [ ] Any specific scalability concerns?

## Testing & Quality

- [ ] What's the minimum test coverage we should target?
- [ ] Do you need E2E tests, or just unit/integration?
- [ ] Any performance benchmarks to hit?

## Deployment & Operations

- [ ] Fly.io deployment - any specific regions?
- [ ] Environment variables/configuration approach?
- [ ] Logging/monitoring requirements?
- [ ] CI/CD pipeline needs?

## Timeline & Phases

- [ ] Should this be built in phases? (Epic breakdown)
- [ ] Which features are MVP vs. nice-to-have?
- [ ] Any hard deadlines?

## Budget

- [ ] Any cost concerns for this project?
- [ ] Should we monitor API/service usage?

---

**Remember**: Ask early, ask often. Clarifying requirements upfront saves time and rework later.
